What is Orbit AI?

Orbit brings together a set of tools that let you work with AI the way you actually need to — not just a chat box, but a full environment for building workflows, connecting data, and running agents that do real work.

At its core, Orbit gives you access to multiple frontier AI models through one interface, with tools built on top: a voice layer, a knowledge container system, multi-agent orchestration, and live browser integration. You can use any of these independently, or combine them into something more powerful.

Orbit is currently in Beta. Things are actively evolving, and some features are still being shaped. If you run into anything unexpected, the Support Center is the best place to start.

Quickstart

You can go from a fresh account to a running agent pipeline in about five minutes. Here's how.

1

Create your account

Head to orbitdev.org and click Join Orbit. Fill in your details, confirm your email, and you're in.

2

Pick a plan

The free Standard Node is a good place to start — you get 1,000 queries to explore the platform. When you're ready for more, upgrading takes seconds. See Pricing & Plans for the full breakdown.

3

Open a session

Log in, choose a model from the Neural Inventory, and start a session through the dashboard or the API. Your first query doesn't need to be anything fancy.

4

Build your first agent

Use the Agent Multi-Build panel to set up a simple workflow. Add a knowledge source, define what the agent should do, and run it. You can always build from there.

Authentication

All API access uses token-based authentication. You'll get a session token when you log in, and every request after that needs to include it.

Getting a token

Send a POST request with your credentials to get started:

bash 
curl -X POST https://api.orbitdev.org/v1/session \
  -H "Content-Type: application/json" \
  -d '{ "email": "you@example.com", "password": "••••••••" }'

The response includes a session_token. Add it to every subsequent request as a Bearer token:

http 
Authorization: Bearer <session_token>

Tokens expire after 24 hours. Use POST /v1/session/refresh before that happens if you need to keep a session alive. Never expose your token in client-side code or public repositories.

Voice Pipeline

The Voice Pipeline lets you stream audio commands to the agent layer in real time. It's designed for situations where typing isn't practical, or where you want voice to be a first-class part of your workflow.

Orbit supports three protocols, each suited to different use cases. WebRTC is the lowest-latency option for live applications. WebSocket works well for server-to-server audio. RTSP relay is available in beta for broadcast-style setups.

Supported protocols

ProtocolTypical LatencyStatus
WebRTC Native Stream< 80msLive
WebSocket Audio< 120msLive
RTSP Relay< 200msBeta

If you're running high-volume voice workloads, the Pro Pulsar Active plan gives you Priority Queue Uplink, which meaningfully reduces wait times under load.

Knowledge Synthesis

Agents are much more useful when they have the right context. Knowledge Synthesis lets you attach external data — documents, repositories, live URLs, or raw runtime data — to any session.

You can mix and match sources. Upload a PDF for background context, connect a Git repo so the agent can see your codebase, and add a live URL that gets refreshed periodically. It all ends up in the same context window.

Supported source types

Agent Multi-Build

This is where things get interesting. The Agent Multi-Build system lets you wire together multiple agents into a single pipeline — each running in its own sandbox, each potentially using a different model.

You can build pipelines visually through the dashboard, or define them in JSON and deploy via the API. Agents can be synchronous or asynchronous, and failures in one node don't cascade to the rest.

json 
{
  "pipeline": "my-research-pipeline",
  "nodes": [
    { "id": "intake",    "model": "propulsar",    "role": "router"    },
    { "id": "synthesize","model": "gpt5_core",    "role": "synthesis" },
    { "id": "output",   "model": "gemini3_ultra", "role": "formatter" }
  ],
  "sandbox": true,
  "async": true
}

Set sandbox: true to isolate each node. Use async: true to run nodes as non-blocking coroutines — helpful when you have independent branches that can run in parallel.

Browser Sync

Browser Sync connects the Orbit agent environment to a live browser session, so your agents can observe, interact with, and respond to real web pages as they change.

This is useful for a wide range of tasks: scraping dynamic content, automating form flows, running web-based tests, or building agents that react to live site state. It works with any modern Chromium-based browser.

Browser Sync requires either the Orbit browser extension or a headless Chromium instance running the Orbit Bridge script. Setup instructions are in the Support Center.

API Reference

The Orbit API is REST-based and returns JSON for all endpoints. Everything in the dashboard can also be done through the API — use whichever fits your workflow.

All requests go to:

base url 
https://api.orbitdev.org/v1

Key endpoints

MethodEndpointWhat it does
POST/sessionStart a new authenticated session
POST/session/refreshExtend a session before it expires
GET/modelsList available models in the Neural Inventory
POST/pipelineCreate and launch an agent pipeline
GET/pipeline/:idCheck the status of a running pipeline
DEL/pipeline/:idStop and remove a pipeline
POST/contextAttach a knowledge source to a session
GET/node/usageSee your current quota and usage

A full interactive API reference and Postman collection are available in the Support Center. For enterprise integrations or custom setup help, reach out through the Architects page.
