Operating model

Small systems, clear ownership.

Orbit is built around visible product surfaces, accountable system boundaries, and reviewable actions. The goal is a platform that feels fast without hiding what it is doing.

Product architecture

Shape chat, files, voice, agents, billing, extension, and desktop flows into one coherent workspace.

Safety architecture

Keep sensitive actions review-first, especially around browser forms, desktop actions, billing, and account controls.

Delivery architecture

Document what exists, ship what works, and keep public pages aligned with the actual app.

Principles

What guides the build.

01
Make the default path obvious.

Users should know where to chat, where to manage accounts, where to ask for help, and where to review actions.

02
Separate public claims from live capabilities.

Docs and marketing should describe implemented behavior, not wishful feature fog.

03
Prefer systems that can be audited.

Agent runs, settings, billing, and support should leave enough context for users and admins to understand what happened.
